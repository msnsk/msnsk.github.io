Godot で作るブロック崩し - Part 1

　

ボールを作る#

前回の Part 1 でプレイヤーが操作するパドルを作ったので、今度はボールを作ります。

ルートノード「Game」の子ノードとして「RigidBody2D」ノードを追加します。

名前を「Ball」としましょう。

次に「Ball」ノードに必要な子ノードを追加していきます。これは「Paddle」ノードの時と同様の作業です。まずは「Sprite」ノードを追加しましょう。

続けて「CollisionShape2D」ノードを追加しましょう。

「Ball」ノードに2つの子ノードが追加されましたね。

それではそれぞれのプロパティを設定していきましょう。まずは「Sprite」ノードから。シーンドックで「Sprite」を選択した状態にして、右側のインスペクタドックを見ると「Texture」プロパティが [空] になっていますね。

その「空」めがけて、ファイルシステムドックから「Ball.png」をドラッグ&ドロップします。

「Texture」プロパティにボールの画像が反映しました。

次に「CollisionShape2D」ノードを選択した状態にして、インスペクタを見ると「Shape」プロパティが [空] になっていますね。

その[空]をクリックして「新規 CircleShape2D」を選択します。

「Shape」プロパティが設定されました。2D ワークスペースを見てみましょう。

半透明の緑色の円が「CollisionShape2D」ノードの形です。サイズが「Sprite」ノードの Texture よりやや大きいので、赤丸のポイントをドラッグしながら大きさを「Sprite」ノードの Texture に合わせましょう。

これくらいにします。「CollisionShape2D」ノードがこれより一回り小さいと 「Sprite」ノードの方が大きくなります。

それではここからインスペクタドックで「Ball」ノードの変更が必要なプロパティを設定していきましょう。シーンドックで「Ball」ノードを選択してインスペクタを見てください。

上から順番にいきましょう。まずはの「Mode」プロパティを変更します。「Character」にします。これによりボールが回転するのを防ぐことができます。

次に「Physics Material」プロパティの [空] をクリックして「新規 PhysicsMaterial」を選択します。

設定した「PhysicsMaterial」をクリックするとさらにプロパティが表示されます。「Friction」プロパティを 0 にして摩擦の影響でボールのスピードが落ちないようにします。また「Bounce」プロパティを 1 に変更して、パドルに当たったらきっちり正確に跳ね返るようにします。

続いて「Gravity Scale」プロパティを設定します。この値は 0 にしないと、ボールが重力の影響を受けて、常に下に落ちる力がかかってしまいます。

「Linear」＞「Damp」プロパティも0にします。これもボールの勢いがなくならないようにするためです。

では 2D ワークスペースで「Ball」ノードを移動させる前に、「Ball」ノードの子を選択不可にしておきましょう。

シーンドックでも選択不可になっていることが確認できました。

ドラッグして「Ball」ノードを移動させる際に、ツールバーでいくつか操作します。

まず、位置をパドルに合わせやすいように、ツールバーから「グリッドスナップ」を有効にします。

そして「移動モード」アイコンをクリックして、ドラッグで移動できるようにします。ショートカットキーを覚えておくと便利です（ Windows: W / macOS: W ）。

2D ワークスペースのディスプレイ表示範囲の左上角（0, 0）から、パドルのちょうど真上にドラッグして「Ball」ノードを動かします。

「Ball」ノードの「Transform」＞「Position」プロパティを確認しておきましょう。x座標が 256 で「Paddle」ノードと同じなので、ボールが今パドルのちょうど真上に位置していることがわかります。もしズレてしまった場合は、プロパティの値を直接触って調整すると効率的です。

ここで一度、シーンを実行してみましょう。

なんだかパドルに対してボールが大きすぎる印象です。サイズを変更しましょう。

シーンドックで「Paddle」ノードの子ノード「Sprite」を選択した状態で、インスペクタの Node2D クラスの「Transform」の「Scale」プロパティで x, y の値をそれぞれ 0.5 とします。サイズを半分にするという意味ですね。

次に同じく「Paddle」ノードの子ノード「CollisionShape2D」ノードを選択して、2D ワークスペース内でポイントをドラッグして、サイズを「Sprite」ノードに合わせましょう。

Memo:
「Paddle」ノードの「Scale」プロパティを直接変更すれば良いのでは？とお思いになる方もいらっしゃるかもしれません。しかし残念ながら、子ノードの「CollisionShape2D」ノードのサイズが変わってなければ、物理エンジンによって「Paddle」ノードの「Scale」プロパティの値が上書きされてしまいます。

ボールがパドルから離れてしまったので、「Ball」ノードを選択した状態にして、2D ワークスペース上でドラッグして移動させましょう。この時、ツールバーでピクセルスナップを有効にして、グリッドスナップは無効にしておくと移動しやすいと思います。

さっきより良いバランスになりましたね！

ボールを発射する#

それではここからスクリプトを作ってボールを動かしていきます。シーンドックで「Ball」ノードを選択した状態で、スクリプトをアタッチします。

先に作っておいた「scripts」フォルダに保存したいので、フォルダアイコンをクリックしましょう。

もしパスが「res://scene」になっていたら、一つ上の階層に戻ってから「scripts」フォルダを選択しましょう。パスが「res://scripts」になったら、ファイル名を「Ball.gd」として「開く」をクリックします。

パスが更新されましたね。「作成」をクリックします。

スクリプトワークスペースが開きました。

今回、スクリプトに記述する全体のコードは以下の通りです。コーディングはちょっと苦手、という方はひとまずこのコードでスクリプトのコードを置き換えてください。

extends RigidBody2D


export (float) var ball_speed = 150.0

func _ready():
	var direction = Vector2(1, -1).normalized()
	var velocity = direction * ball_speed
	apply_impulse(Vector2.ZERO, velocity)

スクリプト解説#

あとで使う_ready()関数以外の部分を削除しましょう。一旦、以下のようになりますね。

extends RigidBody2D



func _ready():
	pass # Replace with function body.

ボールの速さをball_speedという名前のプロパティで定義します。

# ボールの速さ：1秒間に移動する距離（pixel)
export (float) var ball_speed = 150.0

インスペクタで簡単に編集できるようにexportキーワードを頭につけます。(float)はこのプロパティの値が float というデータ型であることを示しています。int 型でも大きな問題はありませんが、今後ボールがだんだん速くなる仕組みを作る時に、より滑らかにスピードアップできた方が良いので float 型にしています。と、散々説明してきてなんですが、値を150.0と浮動小数で記述していることから、GDScript が自動的に float 型と判断するため、(float)の記述は無くても構いません。

次に_ready()関数を編集していきます。まずはpassから始まる行を削除して、別のコードに書き換えていきます。

	pass # Replace with function body.

Memo
funcキーワードで宣言した関数のブロック（インデントで区分されたコードのかたまり）には何かコードを記述しないとエラーになります。まだ記述する内容が決まっていない場合に、一旦passと入れておくことでそのエラーを回避することができます。passにはそれ以上の意味は特にありません。

_ready()関数のブロックに以下のコードを記述します。インデントを忘れないようにご注意ください。

  # ボールが発射される向き
  var direction = Vector2(1, -1).normalized()

_ready()関数内でプロパティを2つ定義しています。

一つ目のプロパティはdirectionです。これはボールが飛んでいく方向を定義しています。Vector2(1, -1)で右上に飛んでいくのはなんとなくおわかりいただけるかと思いますが、.normalized()とはなんでしょう。

まず、座標上で (0, 0) から (1, -1) までの長さが 1 ではないということを理解するところから始めましょう。(1, 0) と (0, -1) はどちらも (0, 0) からの距離が 1 です。その両方の点と接する円があったとすると、その円周に位置する点は全て (0, 0) からの距離が 1 ということになります。しかし、(1, -1) は、その円より外側にありますので、(0, 0) からの距離は 1 より大きい（厳密には √2 ）ということになります。

プロパティdirectionは方向のみを表すプロパティにしたいので、長さを 1 にする必要があります。そこで長さが 1 より大きい (1, -1) の方向はそのままに、長さだけちょうど 1 にしてくれる便利な関数がnormalized()なのです。これは Vector2 クラスの組み込み関数ですので最初から使えます。

次にボールの速度を表すプロパティvelocityです。単純に先に定義した方向の情報を持たない速さを示すプロパティball_speedと方向を示すdirectionの掛け算で定義しています。

  # ボールの速度（方向の情報をもったスピード：方向 x 速さ）
  var velocity = direction * ball_speed

最後に RigidBody2D の組み込み関数apply_impulse()を使って、ボールに衝撃を与えます。

  # 一回だけオブジェクト（ボール）に衝撃を与える
  apply_impulse(Vector2.ZERO, velocity)

これを_ready()関数内に記述したのは、シーンを実行した時にゲームの準備処理段階で1回限りの衝撃をボールに加えてゲーム開始と同時にボールを飛ばすためです。第一引数は offset で不要なためVector2.ZEROで (0, 0) を指定、第二引数は impulse でこれはボールの速度を衝撃として与えるため、先に定義しておいたvelocityを指定しています。apply_impulse()関数について詳しくは Godot 公式ドキュメンテーション をご参照ください。

それでは右上の実行ボタンをクリックして、プロジェクトを実行してみましょう。ショートカットキー操作も便利ですよ（実行 Windows: F5 / macOS: Cmd + B）。

斜め45°の方向へ真っ直ぐ飛んでいきましたね。
