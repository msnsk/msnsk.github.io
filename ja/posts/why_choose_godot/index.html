📔 なぜ Godot を選ぶのか

2021-11-14 · 2021-11-14 · 7 分
  • Godot
  • GodotEngine
  • ブログ
  • ポスト
  • 理由
目次

世の中にはゲームエンジンと呼ばれるゲーム開発用のメインツールがいくつも存在する。中でも Unity や Unreal Engine（以降 UE）の2つは絶大な人気を誇っており、現状、ゲームエンジンの二強となっている。その背景には、プロの開発環境に耐えうる技術仕様により、企業単位で数多く導入され、シェアが拡大し、多くのユーザによる記事や動画でのノウハウがインターネット上に次々と蓄積され、その結果、新たな学習者にとって学びやすい環境となり、さらにまたユーザを増やす、という好循環を生み出しているからだ。他のプログラミング言語でも同じだが、初学者にとって「学習コスト」というのは非常に気になるポイントだ。勉強はできるだけ簡単で少ない方が良い、と思う人が圧倒的多数ではなかろうか。

では、なぜ本 Web サイト「Peanuts Code」では Godot という、日本ではややマイナーなゲームエンジンを紹介しているのか。その10の理由をお伝えしていこうと思う。

1. すっきりとしたユーザーインターフェース

1つ目の理由は、使いやすさだ。Godot は非常に使いやすい。ユーザインターフェースはスッキリとまとまっており、

2. ゲームエンジン内でコーディング可能

コーディングに Godot 専用のプログラミング言語である GDScript を使用するならば、コーディング作業さえも Godot 内でできてしまう。完全に Godot というツールの中だけでゲーム開発を完結することができるため、作業の煩わしさが極めて小さいのだ。Godot はまだ歴史の浅いゲームエンジンだが、それゆえ、他の人気のゲームエンジンからいいとこ取りをして、初期段階から現代のゲーム開発に理想的な形で設計されている。

一方、Unity や UE は、確かに高機能だが、それなりに年月をかけてアップデートを繰り返されて今の形になっており、この2つのゲームエンジンが登場する前に見本になるようなゲームエンジンはなかったであろうから、その構成が若干複雑になってしまうのは無理もないだろう。現に筆者は、初めて Unity で簡単なゲームを作ってみたときに、その UI と開発手順に何とも言えない息苦しさを感じたというのが正直なところだ。また、Unity ではコーディングする際は、C# というプログラミング言語を使うのだが、これを Unity とは別のエディターまたはIDE（例えば Visual Studio ）で行う必要がある。開発中に2つのツールを行き来するのは、どうしても煩わしさを感じるポイントになってしまうのだ。

3. 考え方がオブジェクト指向

また、シーンとノードという基本的なゲームの構成要素を組み合わせ、積み上げていく開発スタイルは、プログラミングのオブジェクト指向そのものであり、非常に合理的で理解しやすい構成になっている。

4. オープンソースだから無料

2つ目の理由は、Godot がオープンソースであることだ。オープンソースの最大のメリットの一つが、それが無料であるということだ。Unity、UE、さらに GameMaker Studio 2（以降 GMS2） も加えて、この3つの人気ゲームエンジンは、企業が商業的に作っているゲームエンジンだ。これら3つのゲームエンジンを使うにはサブスクリプション形式での支払いが生じる。といっても、幸いどれも最初からお金がかかることはなく、Unity、UE一定額の収益が出たら、とか開発したゲームを（商業用に）書き出す必要がある場合はお金がかかる仕組みになっている。最初から売れるゲームを作るなど、誰にもできることではないので、ゲーム開発初心者があまり気にする必要はないが、最終的に売れるゲームを目指して開発するならば、少し気になるところだろう。Godot はオープンソースである限り永遠に無料だ。

5. オープンソースだからユーザ目線

さらにオープンソースのもう一つのメリットは、数人のコアメンバーとそれ以外の有志の Godot ユーザによる開発だ。Godot をより良くするために次々に修正点や追加機能の提案が出され、要望の多いものはどんどん実装されていく。このスタイルによりユーザ目線のツールに磨きがかかっていくのだ。また世界中のユーザが関わっていくため、ローカライズ作業も滞りなく進んでいく。Godot は現時点ですでに日本語にも対応しており、それは Godot というゲームエンジンの UI だけの話ではなく、公式のドキュメンテーションもきっちりとローカライズ作業が進められている。日本語版のドキュメンテーションは、ごく一部英語のままだったりするが、全体を通して比較的わかりやすい翻訳だと筆者は個人的に感じている。

6. 2D / 3D どちらもの開発も可能

一方、Godot は 2D、3D どちらのゲーム開発にも対応しているし、

7. クロスプラットフォーム

使用するためのシステム条件として求められるコンピュータのスペックは高くない。さらにクロスプラットフォームのため、Windows、macOS、Linux どの OS でも利用可能だ。だから Godot なら個人ですぐにゲーム開発を始められるのだ。

8. 個人開発には向いている

現実的な話をすると、ゲーム会社に就職するなら、Unity か UE のスキルを求められることがほとんどだろう。なぜなら、ほとんどの企業がその2つのいずれか、もしくは両方を使ってゲームを開発をしているからだ。特に日本で Godot を使える人の求人などほとんどないだろう。このような状況のため、Godot は個人ゲーム開発で特に利用される傾向が強いと言える。この傾向は今後も長く続くと予想できる。企業がメインの業務ツールを変更するにはかなり工数がかかる。例えば、世の中の業務PCのOSの大半が Windows であり続けることや、世の中の多くのサービスに Java という比較的古参のプログラミング言語が使用し続けられていることにとても似ている。

9. ゲームを個人開発する時代に向いている

筆者が Godot が個人開発向きであることをアピールするのには理由がある。それは筆者が、人生100年時代と言われる昨今、死ぬまでやり続けられる何かがあった方が良いと考えているからだ。その「何か」の選択肢の1つとして、ゲーム開発を非常におすすめなのだ。まず、ゲーム開発を始めるのに年齢は全く関係ない。ゲーム開発をまだしたことがない人には、ちょっとした趣味として一度ゲーム開発にチャレンジしてみて欲しい。ゲーム開発は難しい部分もあるがトータルで楽しい。もしかしたらゲームをするよりも楽しいかもしれない。この「楽しいこと」を通して、プログラミングのスキルを習得できる。さらに、ゲームを作ればそれを販売することもできる（売れるかどうかはさておき）。もしかしたらいつかは大きな副収入にもつながるかもしれない。もし、あなたが毎日暇を持て余しているのであれば、ぜひ一度ゲーム開発にチャレンジしてみてはどうだろうか。何もしない人と毎日何かをし続ける人は少しずつ確実に差が開いていく。一年も経てば大きな差ができる。その差というのはスキルそのものかもしれないし、頭の回転速度や考え方が違ってくるかもしれない。はたまた、収入が大きく変わるかもしれないし、心の充実感や幸福感がもたらされるかもしれない。

10. 海外のコミュニティが活発

一方、Godot の将来性について疑問がある人もいるだろう。先にも述べたが、Godot はまだまだ歴史の浅いゲームエンジンであるが、逆に言えば、まだまだ伸び代のあるこれからが楽しみなゲームエンジンだとも言える。それに、今はまだ日本ではマイナー気味の Godot だが、海外では Godot のコミュニティもかなり活発だ。また、Godot を好んで使用しているユーザたちは大半が Godot を育てることに前向きだ。なぜなら、Godot の将来が彼らの将来にも影響するからだ。特にプロのゲーム開発環境として Godot を使用している人々だ。Godot が長く人気を獲得し続けることで、同じゲームエンジンでの開発を続けることができるし、Godot の人気が落ちて、開発が頓挫してしまえば、別のゲームエンジンに切り替える必要が出てくる。プロの開発者なら、開発途中でメインの開発環境を変えるなんてことは起きないで欲しいと思うに違いない。アップデートも盛んに行われており、現時点で将来消えそうという印象は全くない。そして、筆者自身も Godot を育てたいと願っている。
